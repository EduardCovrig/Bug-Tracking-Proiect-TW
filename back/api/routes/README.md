# Routes (EN)

Routes represent the **final step** in building the backend, where all **endpoints are added to a final router**, thus hiding the concrete implementation details.  

This way, those using the application **can more easily see what interests them**, the endpoints, without needing to know what actually happens behind the scenes.

> To see how the entire data management and retrieval from the databases works, go to [Repositories](../repositories)

> To see how the entire logic part actually works, go to [Services](../services)

> To see all the routes, including the JSON/status sending part for each endpoint, go to [Controllers](../controllers)

# Routes (RO)

Routes reprezinta **ultimul pas** in realizarea backend-ului, in care toate **endpoint-urile se adauga intr-un router final**, ascunzand astfel modul concret de functionare. 

Astfel, cei care utilizeaza aplicatia **pot vedea mai usor ceea ce ii intereseaza**, endpoint-urile, nefiind necesara cunoasterea de ce se intampla efectiv in spate.


> Pentru a vedea cum functoneaza toata partea de gestionare si preluare date din bazele de date, acceseaza [Repositories](../repositories)

> Pentru a vedea efectiv cum functioneaza toata partea de logica, acceseaza [Services](../services)

> Pentru a vedea toate rutele, inclusiv partea de trimitere JSON/status pentru fiecare endpoint, acceseaza [Controllers](../controllers)
