# Services (EN)
Services represent the part that handles all the actual backend logic, such as checks on the received data, format validations, etc.

> To see how the entire data management and retrieval from the databases works, go to [Repositories](../repositories)

> To see all the routes, directly managed by the Router, without seeing the JSON/Status sending part, go to [Routes](../routes)

> To see all the routes, including the JSON/status sending part for each endpoint, go to [Controllers](../controllers)

# Services (RO)
Services reprezinta partea care se ocupa cu toata logica efectiva din spate, precum verificari ale datelor primite, verificari format, etc.

> Pentru a vedea cum functoneaza toata partea de gestionare si preluare date din bazele de date, acceseaza [Repositories](../repositories)

> Pentru a vedea toate rutele, direct gestionate de catre Router, fara a vedea partea de trimitere JSON/Status, acceseaza [Routes](../routes)

> Pentru a vedea toate rutele, inclusiv partea de trimitere JSON/status pentru fiecare endpoint, acceseaza [Controllers](../controllers)
