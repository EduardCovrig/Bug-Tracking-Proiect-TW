# Controllers (EN)

Controllers represent the part that handles the actual management of endpoints, sending **success/error statuses**, along with the **requested JSON**.

> To see all the routes, directly managed by the Router, without seeing the JSON/Status sending part, go to [Routers](../routers)

> To see how the entire logic part actually works, go to [Services](../services)

> To see how the entire data management and retrieval from the databases works, go to [Repositories](../repositories)

# Controllers (RO)
 Controllers reprezinta partea care se ocupa cu gestionarea efectiva a endpointurilor, trimitand catre cel care le acceseaza **statusuri** de reusita/eroare, impreuna cu **json-ul cerut**.
 
> Pentru a vedea toate rutele, direct gestionate de catre Router, fara a vedea partea de trimitere JSON/Status, acceseaza [Routers](../routers)

> Pentru a vedea efectiv cum functioneaza toata partea de logica, acceseaza [Services](../services)

> Pentru a vedea cum functoneaza toata partea de gestionare si preluare date din bazele de date, acceseaza [Repositories](../repositories)
