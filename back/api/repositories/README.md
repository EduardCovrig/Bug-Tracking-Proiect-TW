# Repositories (EN)
Repositories represent the part that handles the actual retrieval of data from the database, without performing additional checks.

> To see all the routes, directly managed by the Router, without seeing the JSON/Status sending part, go to [Routes](../routes)

> To see all the routes, including the JSON/status sending part for each endpoint, go to [Controllers](../controllers)

> To see how the entire logic part actually works, go to [Services](../services)

---

# Repositories (RO)
Repositories reprezinta partea care se ocupa cu preluarea efectiva de date din baza de date, fara a face verificari suplimentare.

> Pentru a vedea toate rutele, direct gestionate de catre Router, fara a vedea partea de trimitere JSON/Status, acceseaza [Routes](../routes)

> Pentru a vedea toate rutele, inclusiv partea de trimitere JSON/status pentru fiecare endpoint, acceseaza [Controllers](../controllers)

> Pentru a vedea efectiv cum functioneaza toata partea de logica, acceseaza [Services](../services)
